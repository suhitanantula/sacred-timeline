## Problem Section

Headline:

> AI can change a project faster than you can understand it.

Copy:

When an AI agent edits your project, the work can move quickly:

- files change across the project
- the working version becomes unclear
- experiments get mixed into the main work
- there is no simple story of what happened
- rollback feels technical or scary

This is not a coding problem. It is a safety problem.

## Solution Section

Headline:

> Sacred Timeline gives AI work a memory and a safety rail.

Copy:

Instead of learning git, you use a few plain-English actions:

| You want to | Say this | What happens underneath |
| --- | --- | --- |
| Save a useful moment | `capture` | A checkpoint is created |
| Try a risky idea | `experiment` | A safe branch is created |
| Keep the idea | `keep` | The experiment becomes part of the main version |
| Abandon the idea | `discard` | The working version stays safe |
| Send work to cloud | `backup` | Your timeline is pushed to GitHub |
| Understand the session | `narrate` | Recent work becomes a plain-English story |

## Installation Section

Headline:

> Start by pasting one prompt into your AI agent.

Copy:

You do not need to understand the terminal. Ask your agent to install and guide you.

Prompt:

```text
Before we change this project, I want to protect it with Sacred Timeline.

I do not know git, so use plain language. Check whether Sacred Timeline is installed, run sacred doctor and sacred status, and tell me whether this project is protected before making changes.
```

Install command:

```bash
curl -fsSL https://raw.githubusercontent.com/suhitanantula/sacred-timeline/main/install.sh | bash
```
