AI agents can change a project faster than you can understand it.

That is powerful, but it creates a new problem. What changed? What still works? How do you go back?

Sacred Timeline is the safety layer for AI-built work.

Before the agent changes anything, run sacred doctor. It checks whether the project is protected, connected to cloud, in sync, and ready for an agent session.

Then run sacred status. Instead of raw git language, you get a simple answer: are we on the main timeline, are there changes, and is the work backed up?

When the agent makes progress, run sacred changes. You can see what moved before you save it.

Then capture the moment in plain English.

No commit flags. No git jargon. Just describe what the agent built.

When you want to try something risky, start an experiment. The working version stays safe while the agent explores a bold direction.

If the experiment works, keep it. If it does not, discard it.

At the end, Sacred Timeline can narrate the session. It turns the work into a story you can understand, share, and continue from.

Then back it up to cloud.

That is the loop: check, capture, experiment, keep or discard, narrate, and backup.

Sacred Timeline is not git for non-developers. It is project safety for people building with AI.

Before the agent changes anything, protect the timeline.
