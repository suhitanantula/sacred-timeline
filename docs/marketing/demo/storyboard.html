Step 1

Check the project before changing anything.

An agent can inspect setup, cloud backup, sync state, and installed skills.

terminal
$ sacred doctor ✓ node: v22.17.0 ✓ safety engine: ready ✓ timeline: active ✓ cloud-connection: origin ✓ sync: ahead=0, behind=0 ✓ codex-skill: installed ✓ claude-skill: installed

Step 2

Know whether the working version is safe.

Plain language replaces technical project state.

terminal
$ sacred status Sacred Timeline Status: ● On main timeline ✓ No uncommitted changes ☁ In sync with cloud
sacred-timeline
Step 3

Let the AI agent work.

When something useful changes, Sacred Timeline can show the difference before you save it.

terminal

Step 4

Capture the useful moment in plain English.

No developer commands. No technical ceremony. Just describe what changed.

terminal
$ sacred capture "create AI work safety landing page" 📸 Captured: "create AI work safety landing page" The working moment is now protected.
sacred-timeline
Step 5

Try the risky idea safely.

Experiments let the agent explore without breaking the working version.

terminal
$ sacred experiment "bolder-demo" 🧪 Experiment "bolder-demo" started. You are now on a safe branch. Main timeline remains protected.
sacred-timeline
Decision point

Keep what works. Discard what does not.

It worked Run sacred keep and the experiment joins the main timeline.
It failed Run sacred discard and the working version stays safe.
You are unsure Ask the agent to explain the change before deciding.
sacred-timeline
Step 6

Turn the session into a story.

Sacred Timeline explains what happened in language a client, founder, or teammate can understand.
